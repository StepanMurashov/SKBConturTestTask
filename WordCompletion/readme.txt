Структура проекта:
- сборка WordCompletions реализует основную логику подбора вариантов автодополнения слов по словарю. Весь внешний интерфейс сборки собран в модуле WordCompletionInterfaces;
- WordCompletionLocalClient - это решение для основной части задачи. Для генерации вариантов автодополнения использует сборку WordCompletions;
- WordCompletionsServer и WordCompletionsNetworkClient - это соответственно сервер и клиент, представляющие собой решение дополнительной части задачи. Для генерации вариантов автодополнения сервер использует сборку WordCompletions.

В задании не был оговорен принцип обработки ошибок. В связи с этим, я принял и использовал следующий принцип:
- если при работе приложения возникает ошибка, то оно завершится с кодом -1;
- дополнительная информация об ошибке выводится методом Trace.TraceError. Соответственно, ее можно увидеть, настроив в конфигурационном файле секцию listeners (system.diagnostics\trace\listeners);
- в клиентских приложениях секция listeners не настроена, поэтому вывод ошибок идет с помощью OutputDebugString, и соответственно увидеть их можно под отладчиком;
- в сервере секция listeners настроена на вывод в консоль.

Папка Tests\ManualTest содержит ряд скриптов для быстрой проверки работы, в том числе:
- LocalTest.bat - запускает локального клиента на обработку приложенного к заданию файла test.in. Результат выводится в файл out.txt;
- StartSever.bat - запускает в отдельном окне сервер со словарем из приложенного к заданию файла test.in;
